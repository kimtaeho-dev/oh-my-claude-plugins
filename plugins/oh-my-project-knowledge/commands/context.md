작업 시작 전에 영향 범위와 비즈니스 규칙을 정리하라.

## 인자 형식

`{도메인}.{기능}` — 수정 대상 기능 (예: poker.투표, jira.인증)

## 코드 지식 로딩

1. `.claude/docs/DOMAIN_MAP.md`에서 해당 기능의 영향 관계 확인
2. `.claude/docs/domains/{domain}/FEATURES.md`에서 상세 영향 관계 + 체크리스트 확인
3. `.claude/docs/DEPENDENCY_MATRIX.md`에서 교차 도메인 영향 확인
4. `.claude/docs/SERVICE_FLOW.md`에서 해당 기능이 포함된 흐름 확인

## 비즈니스 지식 로딩

5. `.claude/docs/business/domains/{domain}/POLICIES.md`에서 해당 기능 ID 섹션의 정책과 에러 시나리오 확인
6. 관련 정책(POL-xxx)과 에러(ERR-xxx)를 나열하라
7. 해당 기능 ID의 코드(FEATURES.md의 코드 위치 참조)를 현재 코드와 비교하여 정책 문서의 내용이 여전히 유효한지 확인하라
